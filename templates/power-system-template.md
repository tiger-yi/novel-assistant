# 力量体系模板 (Power System)

## 1. 核心逻辑 (Core Logic)
- **核心理念**：例如“双修体系”、“科学修仙”（能量与规则运算）等。
- **世界观底层**：能量来源（灵气、煞气）、规则运行方式。

## 2. 境界层级 (Realms)

### A. 体系一（如：武道/身体）
- **境界名**：描述与生理/物理层面的对应（如：炼皮境 - 皮肤韧性强化）。
- **细分**：初期、中期、后期、圆满。

### B. 体系二（如：精神/神识）
- **境界名**：描述与精神/算力层面的对应（如：感知境 - 脑波频率共振）。
- **对应关系**：不同体系间的实力对等逻辑。

## 3. 独特升级路径 (Unique Path)
- **差异化逻辑**：主角与普通人的升级差异（如：量变 vs 硬件迭代/算法升级）。
- **技术映射**：如何用现代/科学逻辑解释传统升级（如：反重力引擎、纳米制程）。

## 4. 核心系统/观想体系 (Core System/Visualization)
- **架构设计**：如“分布式并行计算”、“存算分离”等在识海中的体现。
- **核心组件**：计算节点、存储系统、互连网络（光量子通道）。
- **阶段划分**：构建硬件、部署软件、点亮操作系统（OS）。

## 5. 突破规则 (Advancement Rules)
- **标准条件**：灵气/资源/算力/数据要求。
- **失败后果**：系统崩溃、肉身重创、数据溢出。
- **辅助途径**：升级补丁、驱动优化、灵丹妙药。

## 6. 技法与品阶 (Abilities & Grades)
- **品阶**：凡 / 黄 / 玄 / 地 / 天 / 圣 / 帝 / 禁
- **分类**：身法 / 剑法 / 算法 / 阵法 / 协议 / 神通 / 法宝
- **记录要求**：来源、算力消耗、协议版本、适用境界。

## 7. 战斗规则 (Combat Constraints)
- **数据化战斗**：AR 视角、弱点扫描、轨迹预判、血条显示。
- **超频模式 (Overclocking)**：提升算力/战力的代价（如：硬件损耗、深度昏迷）。
- **越级上限**：基于算力模型定义的越级倍率。

## 8. 术语规范 (Terminology)
- **白名单 (通用)**：灵气、识海、渡劫等传统修仙词汇。
- **主角专用 (科技感)**：算力、众核、接口、底层协议、超频等。
- **黑名单 (禁用)**：禁止出现的词汇（如：西幻词汇、不匹配的度量衡）。

## 9. ReAct 逻辑检查点 (Logic Checkpoints)

> **Thought (推理引导)**
> *   **平衡性**: 当前境界与上一境界的数值差距是否足以形成“碾压”？
> *   **稀缺性**: 突破所需资源（如“天基液”）在 geography.md 中是否有产出逻辑？
>
> **Action (生成规范)**
> *   必须明确每个境界的“特征描写”（如：筑基期可短暂滑翔，金丹期方可御剑）。
> *   定义“越级代价”：若要跨大境界杀敌，必须在 inventory.md 中有对应的代价记录（如：碎裂本命剑）。
>
> **Observation (一致性审计)**
> *   **[Audit]** 检索 characters.toml，检查是否有角色境界超出了当前定义的最高上限。
> *   **[Audit]** 检查正文战斗描写，是否出现了“练气期一拳打爆大乘期”的逻辑崩坏。
