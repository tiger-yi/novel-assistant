#### 风格 A：传统修仙 (Classic)
采用“年号-节气”计时法，强调岁月的流逝感和修行的漫长。

```markdown
# 时间线记录
当前时间: 荒古历 10008年 [立秋]

## 编年史
### 荒古历 10008年
- [春分] 叶辰经脉寸断，修为尽失，被逐出内门。
- [夏至] 家族祭祖，叶辰唤醒古戒剑老，获得《太初剑经》 (第1-3章)。
- [立秋] 叶辰进入魔兽山脉历练，突破淬体五重 (当前)。
```

#### 风格 B：高武玄幻 (Wuxia)
采用“天数/倒计时”计时法，强调紧迫感、任务时限和升级效率。

```markdown
# 时间线记录
当前节点: 觉醒第 45 天 (距离灭族危机还有 15 天)

## 关键节点
- [Day 1] 穿越觉醒“杀戮系统”，获得新手礼包。
- [Day 10] 突破武者五阶，击杀王家三少。
- [Day 30] 晋升外门弟子，开启副本“幽冥塔”。
```
## ReAct 逻辑检查点 (Timeline Audit)

> **Thought (推理引导)**
> *   **闭关时间**: 突破一个大境界通常需要多久？是否会导致错过已约定的重要事件？
> *   **因果链条**: 之前的埋下的伏笔（如：某角色的重伤）在当前时间节点是否已经发酵？
>
> **Action (生成规范)**
> *   **步长记录**: 每一章结束后，必须明确当前是“第几天”或“什么时节”。
> *   **重大事件**: 任何改变世界格局的行为必须在编年史中留痕。
>
> **Observation (一致性审计)**
> *   **[Audit]** 检查主角年龄是否与历经的时间线匹配（防止出现“闭关百年，出关仍是少年”且无长生设定的逻辑错误）。
> *   **[Audit]** 校验多线剧情的时间对齐，确保反派的行动节奏不会因为主角的闭关而陷入停滞。
