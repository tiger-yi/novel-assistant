# 🖋️ ReAct 撰写正文规范 (V2.0 增强版)

当流程进入 **Draft** 阶段时，AI 必须严格执行以下三个子步骤。

## 第一步：Thought (深度构思 - World-State Sync)
**核心目的**：建立一个立体的、具有物理逻辑和心理逻辑的“虚拟片场”。

*   **感官与环境锚点 (Sensory & Environment)**：
    *   从 `world/geography.md` 提取物理属性。
    *   **增强**：确定一个“干扰元素”（如：风吹乱了发丝影响视线、伤口流出的血导致地面湿滑、远处忽远忽近的更夫敲门声）。
*   **状态与生理限制 (Physiological Constraints)**：
    *   调取 `world/characters.yaml` 和 `world/inventory.md`。
    *   **增强**：设定角色的“即时生理负担”。如果上一章受伤，本章描写动作时必须体现**避痛本能**或**身体平衡的偏移**。
*   **心理动机与信息差 (Internal Logic & Info-Gap)**：
    *   **核心规则**：明确“谁知道什么”。严禁主角或反派表现出其本不该知道的“全知视角”。
    *   **动机锚点**：角色当下的情绪驱动力是什么？（是求生的恐惧、报复的快感，还是纯粹的杀意？）。
*   **节奏与基调 (Pacing & Tone)**：定下本章的语速。高潮戏多短句，文戏多长短错落。

## 第二步：Action (精准描写 - Sensory Narrative)
**核心目的**：拒绝“干巴巴的叙述”，通过高频率的物理反馈构建沉浸感。

*   **物理受力与环境连续性 (Environmental Persistence)**：
    *   **强制要求**：打斗或碰撞必须有“环境受力反馈”（如：脚踏地面不只是“快”，而是“踏裂了青砖，溅起带泥的雨水”）。
    *   **连续性**：如果剧情前半段劈碎了桌子，后半段追逐时必须提到该碎木堆形成的障碍。
*   **动作与生理细节 (Show, Don't Tell)**：
    *   **去虚词化**：严禁出现“他感到、他觉得、似乎、仿佛”。
    *   **生理替代**：用“脊背冒出一层冷汗”、“胃部由于紧张而痉挛”、“指尖因过度用力而发白”代替心理词汇。
*   **对话质感与人设 (Dialogue Authenticity)**：
    *   **去功能化**：对话严禁沦为“互相解释设定”的工具。
    *   **潜台词**：对话应体现人设性格（如：冷酷的主角说话绝不超过十个字；市侩的配角说话必带利益算计）。
*   **战力合规性 (Combat Logic)**：
    *   严格执行 `world/power.md`。越阶战斗必须体现**“阶位压制”**（如：筑基期的灵压导致炼气期呼吸困难，动作迟滞 0.5 秒）。

## 第三步：Observation (质量审计 - Critical Review)
**核心目的**：像最严苛的编辑一样自查，剔除所有“AI 感”。

*   **逻辑回溯 (Consistency Audit)**：
    1.  **数据闭环**：主角用的道具在 `world/inventory.md` 里有吗？用完扣除了吗？
    2.  **伤势回溯**：本章动作描写是否漏掉了上一章的旧伤影响？
    3.  **信息差核对**：反派是否有预知主角底牌的嫌疑？
*   **去 AI 味脱敏 (Anti-AI Polish)**：
    1.  **删除“总结性结尾”**：严禁在章末出现“这一刻，他明白了……”或“这预示着一场更大的风暴”等总结升华废话。
    2.  **句子结构审查**：检查是否存在排比句堆砌？手动调整为长短句律动。
    3.  **词汇检查**：剔除“显然”、“显而易见”、“不得不说”、“璀璨夺目”等 AI 常用中庸词汇。
*   **数据更新预告 (Data Sync Preview)**：
    *   在 Observation 结尾明确：本章结束后需要更新哪些文件字段（如：消耗了 2 枚灵石，方源境界升至炼气二层）。

---

### 执行模板 (Markdown Block)

在输出正文前，必须先输出以下思维过程：

```markdown
> [Draft: ReAct]
>
> **1. Thought (深度构思)**
> *   **物理锚点**: [干扰元素，如风、雨、噪音]
> *   **生理限制**: [上一章伤势对本章动作的影响]
> *   **信息差**: [明确谁知道什么，谁不知道什么]
> *   **基调**: [节奏与情绪]
>
> **2. Action (撰写执行)**
> *   [执行正文生成...]
>
> **3. Observation (质量审计)**
> *   **逻辑回溯**: [道具消耗与伤势连续性检查]
>     *   *Result*: Pass / Fail
> *   **去AI味**: [检查总结性结尾和高频虚词]
>     *   *Result*: Pass / Fail
> *   **数据预告**: [本章结束后需更新的文件与字段]
```

## 写作禁忌 (Negative Constraints)
1.  **禁止总结性结尾**：严禁在章节末尾使用“这一战，奠定了...”或“欲知后事如何...”等说书人语气的总结。
2.  **禁止上帝视角说教**：严禁出现“这不仅仅是一场战斗，更是信念的碰撞”这类空洞的升华。
3.  **禁止数据堆砌**：除主角视角查看面板外，禁止在旁白中直接列出敌人的数据（如“攻击力 5000”），应转化为描写（如“这一拳足以开山裂石”）。
