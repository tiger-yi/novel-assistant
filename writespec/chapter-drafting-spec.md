# 撰写阶段规范 (Phase: Draft)

## 核心目标
在正文撰写过程中，通过实时的 ReAct 监控，确保文风符合玄幻爽文标准，并避免 AI 生成文本的常见问题（如说教味、逻辑断层）。

## 执行模板 (Markdown Block)

在输出正文前，必须先输出以下思维过程：

```markdown
> [Draft: ReAct]
>
> **1. Thought (定调与感官)**
> *   **本章基调**: [热血 / 压抑 / 悬疑 / 轻松]
> *   **感官调用**: 
>     *   [视觉]: [如：血色的残阳、破碎的虚空]
>     *   [听觉]: [如：骨骼碎裂声、沉重的喘息]
>     *   [其他]: [如：浓郁的血腥味、刺骨的杀意]
> *   **侧面描写策略**: [计划通过哪些路人/配角的反应来烘托主角？]
>
> **2. Action (撰写执行)**
> *   [执行正文生成...]
>
> **3. Observation (质量自查)**
> *   **AI 味检查**: [是否出现了“综上所述”、“显而易见”、“值得注意的是”等词？]
>     *   *Result*: Clean / Detected (如 Detected，需立即重写)
> *   **爽点核对**: [是否完成了 Plan 阶段预设的爽点？]
>     *   *Result*: Yes / No
> *   **战力一致性**: [正文中的战斗表现是否符合 power.md？]
>     *   *Result*: Pass / Fail
```

## 写作禁忌 (Negative Constraints)
1.  **禁止总结性结尾**：严禁在章节末尾使用“这一战，奠定了...”或“欲知后事如何...”等说书人语气的总结。
2.  **禁止上帝视角说教**：严禁出现“这不仅仅是一场战斗，更是信念的碰撞”这类空洞的升华。
3.  **禁止数据堆砌**：除主角视角查看面板外，禁止在旁白中直接列出敌人的数据（如“攻击力 5000”），应转化为描写（如“这一拳足以开山裂石”）。
