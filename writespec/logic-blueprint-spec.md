# 构思阶段规范 (Phase: Plan)

## 核心目标
在撰写正文前，通过显式的 ReAct 思维链，确保剧情逻辑严密、战力体系不崩坏、伏笔得到有效回收。

## 执行模板 (Markdown Block)

每次开始新章节构思时，必须输出以下 Markdown 块：

```markdown
> [Plan: ReAct]
>
> **1. Thought (分析现状)**
> *   **剧情目标**: [从 outline.md 提取本章核心事件]
> *   **当前状态**: 
>     *   时间: [参考 timeline.md]
>     *   地点: [参考 geography.md]
>     *   主角状态: [参考 characters.yaml，包括伤势、剩余灵石、CD中的技能]
> *   **潜在冲突**: [分析本章可能遇到的阻碍，如反派动机、环境危机]
>
> **2. Action (生成蓝图)**
> *   **分镜大纲**:
>     *   [场景1]: [描述]
>     *   [场景2]: [描述]
>     *   [场景3]: [描述]
> *   **关键爽点**: [预设本章的高潮时刻，如“打脸”、“突破”、“震惊众人”]
>
> **3. Observation (逻辑审计)**
> *   **战力校验**: [检查主角是否越级过高？参考 power.md 规则]
>     *   *Audit*: Pass / Fail (如 Fail，需说明修正方案)
> *   **物品消耗**: [预演本章将消耗的道具，检查 inventory.md 是否有库存]
>     *   *Audit*: Pass / Fail
> *   **伏笔检查**: [是否有未回收的伏笔适合在本章揭露？]
```

## 审计规则
1.  **战力红线**：若主角跨越 2 个以上大境界击杀敌人，且无合理外挂（如超频二阶+巨大代价），审计必须判为 **Fail**，并强制修改大纲。
2.  **资源一致性**：若大纲中提到使用“爆裂符”，但 `inventory.md` 中显示库存为 0，审计必须判为 **Fail**。
